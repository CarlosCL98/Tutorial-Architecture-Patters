DataBase Tutorial


Amazon Relational Database Service (Amazon RDS), permite de una manera sencilla configurar, utilizar y escalar una base de datos relacional en la nube. El servicio suministra capacidad rentable y escalable al mismo tiempo que automatiza las arduas tareas administrativas, como el aprovisionamiento de hardware, la configuración de bases de datos, la implementación de parches y la creación de copias de seguridad. Los siguientes son los pasos para poder enlazar a nuestro formulario dinámico creado anteriormente a una base de datos usando este servicio:

  • En la consola de AWS buscar el servicio RDB (Relational DataBase). NOTA: TAMBIÉN SE PUEDE REALIZAR ESTE PROCEDIMIENTO CON UNA BASE DE DATOS NO RELACIONAL COMO MONGODB.
  • Ahora procederemos a crear una nueva base de datos haciendo click en create database. Acontinuación pueden elegir el método para crear una base de datos estándar o de fácil creación. En este tutorial se seguirá el estándar. Se selecciona la base de datos a usar, la versión y más adelante el nombre de la base de datos, el usuario y la contraseña para su acceso. NOTA: PARA QUE SEA POSIBLE CONECTARSE DESDE LA MÁQUINA EC2, LA BASE DE DATOS DEBE TENER EL MISMO GRUPO DE SEGURIDAD QUE LA MÁQUINA.
  • Luego de haberla creado, accedemos a ella y copiamos el endpoint que nos permitirá la conexión. Ahora configuramos en nuestro proyecto dicha conexión para alamcenar los datos que necesitamos.
  • Después de haber verificado la conexión y que la base de datos ya almacene los datos ingresados en el formulario, entonces volvemos a general el .JAR de nuestro proyecto y lo subimos a la máquina EC2 como se indicó en el anterior tutorial.
  • Ahora nos conectamos por medio de ssh y arrancamos nuestro proyecto.
  • En seguida ya podemos acceder con la dirección IP pública o DNS público de nuestra máquina acompañado con el puerto correspondiente, llenar el formulario y nuestra base de datos ya debería tener los datos almacenados del formulario.
