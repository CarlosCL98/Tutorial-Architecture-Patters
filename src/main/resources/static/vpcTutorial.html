VPC Tutorial


Amazon Virtual Private Cloud (Amazon VPC) permite aprovisionar una sección de la nube de AWS aislada de forma lógica, en la que puede lanzar recursos de AWS en una red virtual que usted defina. Puede controlar todos los aspectos del entorno de red virtual, incluida la selección de su propio rango de direcciones IP, la creación de subredes y la configuración de tablas de ruteo y gateways de red. Puede usar tanto IPv4 como IPv6 en su VPC para obtener acceso a recursos y aplicaciones de manera segura y sencilla. Los siguientes son los pasos para poder crear VPCs de manera que sea posible crear redes y subredes y dar a nuestra máquina una dirección ip de rango propio y hacer un mejor seguimiento:

  • En la consola de AWS buscar el servicio VPC y accedemos a su consola de administración.
  • Ahora vamos a usar el asistente para crear una VPC con una sola subnet pública.
  • Ahora debemos darle los rangos de las direcciones IP que queremos. Para la red general el rango debe estar hasta direcciones IP con máscara /16, pero para la subred es posible hasta /24. Les asignamos un nombre y finalmente la creamos.
  • En la consola de VPC podemos crear más subredes si tenemos una red muy amplia o si deseamos separar mejor los componentes y dispositivos que tenemos.
  • En seguida pasamos a crear una tabla de enrutamiento y le añadimos las rutas como por ejemplo del gateway, con el fin de ser capaces de comunicarnos entre subredes. Para esto también deben agregarse las subredes que queremos que puedan entrar y salir con peticiones.
  • Para probar que la VPC y las subredes asignen direcciones IP públicas y esté funcionando, se puede crear una instancia en EC2 y asignarle una subred, indicando que automáticamente asigne las direcciones IP.
