EC2 Tutorial


Amazon Elastic Compute Cloud (Amazon EC2) es un servicio web que proporciona capacidad informática en la nube segura y de tamaño modificable. Está diseñado para simplificar el uso de la informática en la nube a escala web para los desarrolladores. Los siguientes son los pasos para poder desplegar un formulario dinámico usando este servicio:

  • En la consola de AWS busco el servicio EC2. Al hacer click sobre él, se redireccionará hacia el escritorio EC2.
  • En seguida, haga click sobre Launch Instance y selecciona la imagen de una instancia de Amazon que más desee y se ajuste a sus necesidades. En este caso seleccionaremos una máquina Linux (puede ser Amazon Linux AMI o Ubuntu) que contiene lo necesario para realizar el despliegue de nuestro formulario dinámico. NOTA: PUEDE REQUERIR LA INSTALACIÓN DE COMPONENTES COMO MAVEN O UNA VERSIÓN SUPERIOR DE JAVA.
  • Ahora se selecciona la opción Free tier eligible que ya viene marcada por defecto y se selecciona review and launch.
  • En esa página se dirige a editar security group. Ahí se activa HTTP y HTTPS para permitir el tráfico web. Finalmente se da click en launch.
  • En la siguiente ventana se pedirá crear o usar un par de llaves para poder acceder a la instancia. Si no tiene una creada, cree un nuevo par con el nombre deseado y descargue las llaves almacenándolas en un sitio seguro.
  • Después de unos momentos el estado de nuestra instancia será running, de manera que ya es posible acceder a ella.
  • Ahora, debemos crear un nuevo proyecto que permita ejecutar un servicio web. Ahí crearemos los componentes necesarios para hacer un formulario dinámico.
  • Después de haber creado el proyecto y verificado que el formulario funciona correctamente, procedemos a conectarnos mediante sftp a nuestra máquina para cargar el proyecto o el .JAR respectivo. NOTA: PARA REALIZAR ESTA CONEXIÓN DEBEMOS ESTAR SOBRE LA RUTA EN LA QUE SE ENCUENTRA NUESTRO PAR DE LLAVES
  • Después de haber subido el archivo a la máquina en AWS, procedemos a conectarnos mediante ssh y así ejecutar nuestro proyecto. NOTA: EL PUERTO DONDE EL PROYECTO SERÁ DESPLEGADO DEBE COINCIDIR CON EL DEL GRUPO DE SEGURIDAD CONFIGURADO.
  • Finalmente, accedemos con la dirección IP pública o DNS público de nuestra máquina acompañado con el puerto correspondiente y nuestro formulario ya debería aparecer.
